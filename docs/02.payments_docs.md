# 결제 모듈 프로젝트

## 프로젝트 개요

- 본 프로젝트는 쇼핑몰 서비스 내에서 사용자가 카드를 등록하고, 등록된 카드로 결제를 진행할 수 있도록 설계된 결제 모듈입니다.

### 주요 기능
- 카드 등록 기능
- 카드 번호 마지막 8자리 마스킹 처리
- 카드 정보 유효성 검사
- 카드 프리뷰 실시간 반영
- 보유 카드 목록 표시
- 선택 카드 결제 기능 (Mock 결제)
- 입력칸 UI 시안 반영 (회색 박스 스타일)
- 닫기 버튼 및 화면 전환 구조

### 프로젝트 목표
- 실제 카드 결제 UX 흐름과 유사한 구조 구현
- 상태 관리(Context API)를 활용한 카드 데이터 관리
- 보안 요구사항(마스킹, CVC 미저장) 반영
- 시안 기반 UI 재현

### 참여 팀원
- 완두앙금빵


## 개발 환경
- React 18
- JavaScript (ES6+)
- Tailwind CSS
- Context API (상태 관리)
- Vite (번들러)
- Git / GitHub

## 설치 및 실행 방법

### 1. 프로젝트 클론
```
git clone https://github.com/Joo-HY/Shooking.git
```
### 2. 패키지 설치
```
npm install
```
### 3. 개발 서버 실행
```
npm run dev
```
### 4. 배포 버전 빌드
```
npm run build
```
## 컴포넌트 구조

```
App
 └ PaymentModule
     ├ CardListView
     │   └ SavedCardItem
     └ PaymentForm
         ├ CardPreview
         ├ CardNumberInput
         ├ ExpiryInput
         ├ CardHolderInput
         └ CvcInput
```

### 1. PaymentModule
- 결제 모듈의 최상위 컨테이너
- 카드 목록 화면 / 카드 추가 화면 전환 관리
- 결제 실행(Mock) 처리

### 2. CardListView
- 등록된 카드 목록 표시
- 카드 선택 기능
- “이 카드로 결제하기” 버튼 제공
- 카드가 없을 경우 안내 문구 표시
- 우측 상단 닫기 버튼 구현

### 3. SavedCardItem
- 카드 UI 프리뷰 컴포넌트
- 카드번호 표시 형식 :
```
1111 2222 •••• ••••
```

### 4. PaymentForm
- 카드 추가 입력 폼
- 유효성 검사
- 작성 완료 시 카드 저장(Context)

### 5. CardNumberInput
- 카드번호 입력 전용 컴포넌트
- 16자리 숫자만 허용
- 9번째 자리부터 3~4그룹 마스킹
- Overlay 방식으로 구현 (입력과 표시 분리)
- 붙여넣기(paste) 처리 지원

### 6. ExpiryInput
- MM/YY 형식 자동 포맷
- 1~12월 범위 제한

### 7. CardHolderInput
- 영문 대문자 + 공백만 허용
- 최대 30자 제한
- 카드 표기 이름과 동일 입력 유도

### 8. CvcInput
- 숫자 3~4자리 허용
- 입력 시 마스킹 처리
- 저장하지 않음

### 9. CardsContext
- 카드 상태 전역 관리
- addCard
- selectCard
- selectedCard
- CVC는 저장하지 않음

## 테스트 방법

### 수동 테스트 케이스
1. 카드번호 입력 테스트
- 8자리 이하 입력 시 그대로 표시
- 9자리 이상 입력 시 3~4그룹 마스킹
- 16자리 초과 입력 불가
- 붙여넣기 정상 동작

2. 만료일 테스트
- 00 입력 시 01로 보정
- 13 입력 시 12로 보정
- 4자리 미만 입력 시 에러 표시

3. 카드 소유자 이름
- 한글 입력 시 자동 제거
- 소문자 입력 시 자동 대문자 변환
- 2자 미만 입력 시 에러

4. CVC 테스트
- 숫자 외 입력 불가
- 3자리 미만 시 에러

## 유의사항 및 알려진 이슈
- 실제 결제 API는 연결되어 있지 않으며 Mock 처리 상태입니다.
- 카드번호 input은 overlay 방식으로 구현되어 있으며, 커서 위치는 항상 끝 기준입니다.
- 모바일 브라우저에서 키보드 자동완성 기능은 비활성화되어 있습니다.
- 실제 서비스에서는 카드번호 전체를 클라이언트에 저장하지 않아야 합니다.

## 추가 항목

### 설계의도
- 카드번호 마스킹은 보안 요구사항을 반영하여 표시 레이어와 입력 레이어를 분리하는 구조로 구현했습니다.
- Context API를 사용하여 카드 상태를 전역에서 관리하도록 설계했습니다.
- 시안 기반 UI 구현을 위해 Tailwind 기반 레이아웃을 통일했습니다.

- 카드 삭제 기능
- 카드 다중 관리 UX 개선
- 커서 위치 유지 고도화
- E2E 테스트 도입
