# 장바구니 프로젝트

## 프로젝트 개요

본 프로젝트는 쇼핑몰 웹 애플리케이션에서 사용자가 선택한 상품을 확인하고 주문할 수 있도록 하는 장바구니 기능을 구현하는 것을 목표로 한다. 사용자는 장바구니 페이지에서 담은 상품 목록을 확인하고, 상품의 수량을 조절하거나 삭제할 수 있으며 최종 결제 단계로 이동할 수 있다.

또한 본 프로젝트에서는 컴포넌트 기반 구조를 적용하여 장바구니 UI를 구성하고, Storybook을 활용하여 주요 UI 컴포넌트를 문서화하였다. 이를 통해 컴포넌트의 재사용성을 높이고 협업 시 UI 명세를 명확하게 전달할 수 있도록 하였다.

주요 기능은 다음과 같다.

- 장바구니에 담긴 상품 목록 표시
- 상품 이미지, 상품 정보, 가격 표시
- 장바구니 상품 수량 관리
- 상품 삭제 기능
- 장바구니 상태에 따른 결제 버튼 활성화/비활성화
- Storybook을 활용한 UI 컴포넌트 문서화

참여한 팀원
- 완두앙금빵

## 개발 환경

- React
- JavaScript (ES6)
- Vite
- Tailwind CSS

## 설치 및 실행 방법

```
npm install
npm run dev
```

## 컴포넌트 구조

장바구니 페이지는 여러 개의 UI 컴포넌트로 구성되어 있으며, 각 컴포넌트는 역할에 따라 분리되어 관리된다.

### CartPage
장바구니 페이지의 전체 레이아웃을 담당하는 상위 컴포넌트이다. 장바구니에 담긴 상품 목록과 결제 영역을 포함한다.

### CartItem

장바구니에 담긴 개별 상품 정보를 표시하는 컴포넌트이다. 상품 이미지, 상품명, 가격 등의 정보를 보여준다.

### CartItemList

장바구니에 담긴 상품 목록을 관리하고 여러 개의 CartItem 컴포넌트를 렌더링한다.

### CheckoutButton

사용자가 장바구니에 담긴 상품을 결제 단계로 진행할 수 있도록 하는 버튼 컴포넌트이다. disabled prop을 통해 버튼의 활성화 상태를 제어할 수 있으며, 클릭 시 결제 로직 또는 결제 페이지 이동 이벤트를 실행한다.

### EmptyCart

장바구니에 담긴 상품이 없을 경우 사용자에게 빈 상태 UI를 보여주는 컴포넌트이다.

## 테스트 방법
프로젝트에는 기본적인 테스트 환경이 구성되어 있으며, 테스트 명령어를 통해 테스트를 실행할 수 있다.
```
npm test
```
또한 Storybook을 활용하여 컴포넌트의 다양한 상태를 시각적으로 확인하고 UI 동작을 검증할 수 있다.

예를 들어 CheckoutButton 컴포넌트의 경우 다음과 같은 상태를 확인할 수 있다.

- 기본 상태 (Default)
- 비활성화 상태 (Disabled)

이를 통해 컴포넌트의 동작과 UI 상태를 쉽게 테스트할 수 있다.

## 유의사항 및 알려진 이슈

현재 프로젝트에서 확인된 사항은 다음과 같다.

- 장바구니 데이터는 현재 프론트엔드 상태 기반으로 관리되고 있으며, 실제 서버 API와 연동되지는 않았다.

- 결제 기능은 실제 결제 시스템과 연결되지 않은 UI 수준의 기능으로 구현되어 있다.

- 장바구니 데이터가 새로고침 시 초기화되는 문제가 있으며, 향후 LocalStorage 또는 서버 API 연동을 통해 개선할 예정이다.

## 추가 항목

Storybook 문서화

본 프로젝트에서는 UI 컴포넌트의 구조와 동작을 명확히 하기 위해 Storybook을 활용하여 컴포넌트를 문서화하였다.

대표적으로 다음 컴포넌트를 문서화하였다.

- CheckoutButton

Storybook을 통해 다음과 같은 정보를 확인할 수 있다.

- 컴포넌트의 props
- UI 상태별 렌더링 결과
- 컴포넌트 동작 예시

이를 통해 개발 과정에서 컴포넌트의 재사용성과 유지보수성을 높일 수 있다.
