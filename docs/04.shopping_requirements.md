# 쇼핑몰 연동 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
슈킹 쇼핑몰 연동 

### 사용하게 될 기술: 
- React (Vite 기반)
- JavaScript (ES6+)
- React Router DOM (페이지 라우팅)
- Context API (전역 상태 관리)
- Tailwind CSS (스타일링)
- LocalStorage (데이터 유지, 선택 적용)

### 문서 작성자: 
완두앙금빵

## 📝 고객 요구사항 정리
### 1. 페이지 간 기능 연동
기존에 개별적으로 구현된 상품 목록, 장바구니, 결제 페이지를 하나의 흐름으로 연결해야 한다.
사용자는 자연스럽게 아래의 흐름을 따라 이동할 수 있어야 한다.

- 상품 목록 → 상품 상세 페이지 이동
- 상품 상세 페이지 → 장바구니 담기
- 장바구니 → 결제 페이지 이동
- 결제 완료 → 결제 완료 페이지 이동
- 결제 완료 페이지 → 상품 목록으로 복귀

각 페이지는 단순 이동이 아니라 이전 상태를 유지한 채로 연결되어야 한다.

### 2. 장바구니 데이터 일관성 유지

사용자가 어느 페이지에서 상품을 담거나 이동하더라도 장바구니 데이터는 유지되어야 한다.

구체적으로는 다음을 만족해야 한다.

- 상품 목록에서 담은 상품이 장바구니 페이지에 반영
- 상품 상세 페이지에서 담은 상품도 동일하게 반영
- 페이지 이동 시 장바구니 데이터가 사라지지 않음
- 동일 상품 추가 시 수량이 누적되도록 처리

또한 결제 페이지에서는 장바구니 데이터를 기반으로

- 총 상품 개수
- 총 결제 금액

이 정확하게 계산되어야 한다.

3. 상품 상세 페이지 신규 추가

기존에 없던 상품 상세 페이지를 새롭게 구현해야 한다.

상품 상세 페이지에는 다음 정보와 기능이 포함되어야 한다.

- 상품 이미지
- 브랜드명
- 상품 설명
- 가격
- 수량 선택 기능 (+ / -)
- 장바구니 담기 버튼

또한 추가 요구사항으로,

- 동일 브랜드의 다른 상품을 “관련 상품” 영역에 표시해야 한다.
- 관련 상품 클릭 시 해당 상품의 상세 페이지로 이동해야 한다.

### 4. 결제 완료 페이지 구성

결제 완료 후 사용자에게 결과를 명확하게 보여주는 페이지가 필요하다.

포함되어야 할 정보는 다음과 같다.

- 결제 완료 메시지
- 구매한 총 상품 개수
- 총 결제 금액
- 상품 목록으로 돌아가는 버튼

결제 완료 페이지는 단순 안내가 아니라 결제 결과를 요약 제공하는 화면이어야 한다.

5. 사용자 경험(UX) 요구사항

고객사는 전문적인 용어보다는 직관적인 사용 경험을 요구하고 있다.

따라서 다음 사항을 고려해야 한다.

- 버튼과 기능은 직관적으로 이해 가능해야 함
- 페이지 이동 시 끊김 없는 흐름 유지
- 상품 정보 및 가격이 명확하게 표시되어야 함
- 관련 상품 추천은 자연스럽게 노출

## 📋 기능 목록

### 1. 라우팅 기능
- 상품 목록 페이지 라우팅 (/ 또는 /products)
- 상품 상세 페이지 라우팅 (/products/:id)
- 장바구니 페이지 라우팅 (/cart)
- 결제 페이지 라우팅 (/payment)
- 결제 완료 페이지 라우팅 (/payment/complete)

### 2. 상품 목록 기능
- 상품 리스트 렌더링
- 상품 클릭 시 상세 페이지 이동
- 장바구니 담기 기능 (선택 구현)

### 3. 상품 상세 페이지 기능
- URL 파라미터 기반 상품 조회
- 상품 이미지, 브랜드, 가격, 설명 표시
- 수량 증가/감소 기능
- 장바구니 담기 기능
- 관련 상품 리스트 표시 (동일 브랜드 기준)
- 관련 상품 클릭 시 상세 페이지 이동

### 4. 장바구니 기능
- 장바구니 상품 목록 표시
- 상품별 수량 관리
- 동일 상품 추가 시 수량 증가 처리
- 총 상품 개수 계산
- 총 결제 금액 계산
- 결제 페이지로 이동 기능

### 5. 결제 기능
- 장바구니 데이터 기반 주문 정보 표시
- 결제 진행 버튼
- 결제 완료 시 페이지 이동 처리

### 6. 결제 완료 기능
- 총 구매 상품 수 표시
- 총 결제 금액 표시
- 결제 완료 메시지 출력
- 상품 목록 페이지로 이동 버튼

### 7. 상태 관리 기능
- 장바구니 전역 상태 관리 (Context API)
- 상품 수량 및 총액 자동 계산
- 페이지 간 데이터 공유

### 8. 데이터 유지 기능 (선택)
- LocalStorage를 활용한 장바구니 데이터 저장
- 새로고침 시 데이터 복원

### 9. UI/UX 기능
- 수량 조절 버튼 인터랙션
- 장바구니 담기 버튼 피드백
- 결제 완료 후 명확한 안내 화면 제공
- 관련 상품 추천 UI 구성
