# 결제 모듈 리뷰

## 고객사 요구사항
### 1. 결제 화면 제공
사용자가 상품을 구매할 때 별도의 페이지로 이동하지 않고 현재 화면에서 바로 결제를 진행할 수 있도록 팝업 형태의 결제 화면을 제공했습니다.
이를 통해 사용자는 상품 페이지를 벗어나지 않고 자연스럽게 결제 과정을 진행할 수 있습니다.
### 2. 카드 정보 입력 기능
사용자가 결제에 사용할 카드 정보를 입력할 수 있도록 다음과 같은 입력 항목을 제공합니다.
- 카드 번호
- 카드 유효기간
- CVC 번호

입력 과정에서 사용자가 편하게 정보를 입력할 수 있도록 다음 기능을 포함했습니다.
- 카드 번호 입력 시 자동으로 구분이 적용되어 읽기 쉽게 표시
- 숫자만 입력되도록 제한하여 입력 오류 방지
- 입력 필드마다 안내 문구 제공

### 3. 입력 정보 확인 기능
사용자가 결제를 진행하기 전에 입력된 카드 정보가 올바른지 확인하는 기능을 구현했습니다.
다음과 같은 경우에는 결제가 진행되지 않도록 안내합니다.
- 카드 번호가 올바른 형식이 아닌 경우
- 카드 유효기간이 입력되지 않았거나 형식이 맞지 않는 경우
- CVC 번호가 입력되지 않은 경우

이 경우 화면에 안내 메시지를 표시하여 사용자가 쉽게 수정할 수 있도록 했습니다.

### 4. 결제 진행 기능
모든 입력 정보가 정상적으로 입력된 경우 결제를 진행할 수 있습니다.
결제 결과에 따라 다음과 같은 안내를 제공합니다.
- 결제가 정상적으로 완료된 경우: 결제 완료 메시지 표시
- 결제 처리 중 문제가 발생한 경우: 결제 실패 안내 메시지 표시

이를 통해 사용자가 현재 결제 상태를 명확하게 확인할 수 있습니다.

### 5. 카드 정보 보호 기능
사용자의 카드 정보 보호를 위해 카드 번호를 그대로 노출하지 않도록 처리했습니다.
예를 들어 카드 번호는 다음과 같이 표시됩니다.
```
1234 5678 9012 3456 → 1234 5678 **** ****
```
이 방식은 카드 번호의 일부만 표시하여 사용자 정보 보호와 카드 식별의 편의성을 동시에 고려한 방식입니다.

## 요구사항에 따른 기능 동작 확인
### 1. 결제 화면 실행
사용자가 상품 페이지에서 결제 버튼을 클릭하면 결제 화면이 나타납니다.

결제 화면은 다음과 같은 특징을 가지고 있습니다.
- 현재 페이지 위에 팝업 형태로 표시됨
- 결제에 필요한 정보 입력 영역 제공
- 취소 또는 닫기 버튼을 통해 언제든 결제 화면을 닫을 수 있음

이를 통해 사용자가 결제 진행 여부를 쉽게 선택할 수 있습니다.

### 2. 카드 정보 입력
결제 화면에서는 카드 결제에 필요한 정보를 입력할 수 있습니다.

#### 카드 번호 입력

사용자가 카드 번호를 입력하면 숫자가 일정 단위로 자동 정리되어 표시됩니다.

예시
1234567812345678 입력 → 1234 5678 1234 5678

이 기능은 카드 번호를 더 쉽게 확인할 수 있도록 돕습니다.

#### 유효기간 입력

카드의 유효기간을 입력할 수 있는 입력란을 제공합니다.

입력 형식이 맞지 않거나 값이 비어 있는 경우 사용자에게 안내 메시지를 제공합니다.

#### CVC 입력

카드 뒷면에 있는 CVC 번호를 입력하는 영역을 제공합니다.

이 값은 보안을 위해 입력 시 화면에 그대로 노출되지 않도록 처리합니다.

### 3. 입력 오류 안내

사용자가 잘못된 정보를 입력했을 경우 즉시 확인할 수 있도록 안내 기능을 제공했습니다.

다음과 같은 상황에서 안내 메시지가 표시됩니다.

- 카드 번호가 올바른 형식이 아닐 때
- 필수 입력 항목이 비어 있을 때
- 유효기간 형식이 맞지 않을 때

사용자는 안내 메시지를 확인하고 입력 정보를 수정할 수 있습니다.

### 4. 결제 진행 과정

모든 정보가 정상적으로 입력되면 사용자는 결제 버튼을 눌러 결제를 진행할 수 있습니다.

결제 진행 과정은 다음과 같습니다.

1. 입력된 카드 정보 확인
2. 결제 요청 처리
3. 결제 결과 안내

결제 처리 결과는 사용자에게 명확하게 표시됩니다.

### 5. 결제 결과 안내

결제 요청이 처리되면 다음과 같은 결과 안내가 표시됩니다.

#### 결제 성공

결제가 정상적으로 완료되면 사용자에게 결제 완료 메시지가 표시됩니다.

사용자는 결제가 성공적으로 처리되었음을 확인할 수 있습니다.

#### 결제 실패

결제 처리 중 문제가 발생한 경우 결제 실패 안내 메시지가 표시됩니다.

이 경우 사용자는 다음과 같은 행동을 할 수 있습니다.

- 카드 정보를 다시 입력
- 결제를 다시 시도

6. 카드 정보 표시 방식

사용자의 카드 정보 보호를 위해 카드 번호는 일부 숫자를 가려 표시됩니다.

예시

1234 5678 9012 3456
→
1234 5678 **** ****

이 방식은 다음과 같은 목적을 가지고 있습니다.

- 사용자 카드 정보 보호
- 다른 사람이 화면을 보더라도 카드 번호 전체가 노출되지 않도록 방지
- 사용자가 자신의 카드인지 확인할 수 있도록 일부 숫자는 표시

7. 사용자 편의성 고려 사항

이번 결제 모듈 구현에서는 사용자의 편의성을 고려하여 다음과 같은 점을 반영했습니다.

- 결제 화면을 별도 페이지가 아닌 팝업으로 제공
- 카드 입력 형식 자동 정리
- 잘못된 입력에 대한 안내 메시지 제공
- 결제 성공 및 실패 상태를 명확하게 안내

이를 통해 사용자가 복잡한 과정 없이 자연스럽게 결제를 진행할 수 있도록 설계했습니다.
