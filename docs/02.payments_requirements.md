# 결제 모듈 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
슈킹 쇼핑몰 결제 모듈

### 사용하게 될 기술: 
- 프레임워크 : React
- 빌드 툴 : Vite
- 언어 : JavaScript
- 스타일링 : Tailwind CSS
- 테스팅 : Vitest
- 패키지 매니저 : npm
- Version Control : Git / GitHub
- Deployment : GitHub Pages

### 문서 작성자: 
완두앙금빵

## 📝 고객 요구사항 정리
### 1) 카드 등록 기능
- 사용자는 새로운 결제 카드를 등록할 수 있어야 한다
- 카드 등록은 카드 입력 화면(모달 또는 페이지)에서 진행한다
- 사용자는 여러 개의 카드를 등록할 수 있다

### 2) 카드 민감 정보 보안 처리 (필수)
- 카드번호 보안
  - 카드번호 전체 노출 금지
  - 카드 목록 및 결제 화면에서는 반드시 마스킹 적용:
```
1111 - 2222 - •••• - ••••
```
- 카드번호 마지막 8자리
  - 별도 입력값이 아니라 마스킹 처리 대상 정보
  - UI에 직접 노출 금지
- 보안코드 (CVC/CVV)
  - 입력 시 항상 마스킹 처리
  - 평문 노출 금지
- 카드 비밀번호
  - 일부 자리 입력
  - 입력값 마스킹 처리

### 3) 구매 버튼 기능 요구사항
- 상품 목록 카드 UI에:
  - 기존 "담기" 버튼 유지
  - "구매" 버튼 추가
- 구매 버튼 클릭 시:
  - 결제 모듈 진입
  - 카드 선택 또는 카드 등록 화면 표시

## 📋 기능 목록
### 1) 카드 등록 기능 요구사항
- 카드 등록 UI 제공
- 카드 정보 입력 후 저장
- 등록 카드 리스트 관리
```
<AddCardModal />
<CardForm />
```
 
### 2) 카드 민감 정보 보안 처리 요구사항
- 카드번호 마스킹
- 카드번호 마지막 8자리 노출 금지
- CVC 입력 마스킹
- 카드 비밀번호 입력 마스킹
```
maskCardNumber()
maskSensitiveInput()
```

### 3) 구매 버튼 기능 요구사항
```
<PurchaseButton />
<ProductActions />
```
기능:
  - 상품 카드에 구매 버튼 추가
  - 클릭 시 결제 흐름 시작

### 4) 카드 번호 입력 기능 요구사항
- 숫자만 입력 허용
- 자동 포맷팅 (4자리 단위)
- 입력값 검증
- 저장 시 마스킹 처리
```
<CardNumberInput />
```

### 5) 만료일 입력 기능 요구사항
- MM/YY 형식
- 숫자 입력만 허용
- MM 범위 검증 (01~12)
- 만료된 날짜 입력 제한

```
<ExpiryInput />
```

### 6) 카드 소유자 입력 기능 요구사항
- 문자열 입력
- 길이 제한
- 카드 표기 이름과 동일 입력 안내

```
<CardHolderInput />
```

### 7) 보안코드(CVC/CVV) 입력 기능 요구사항
- 숫자 입력
- 입력값 마스킹
- 길이 제한 (3~4자리)

```
<CvcInput masked />
```

### 8) 카드 비밀번호 입력 기능 요구사항
- 숫자 입력만 허용
- 일부 자리 입력
- 입력값 마스킹 처리

```
<CardPasswordInput />
```
