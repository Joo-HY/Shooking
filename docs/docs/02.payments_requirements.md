## 📝 고객 요구사항 정리
### 1) 결제 흐름 요구사항
- 상품 목록에서 ‘담기’ 버튼 옆에 ‘결제’ 버튼 추가
- 결제 버튼 클릭 시 결제 모듈 진입
- 사용자 카드 선택 후 결제 진행

### 2) 카드 관리 기능 요구사항
- 사용자는 여러 개의 카드 등록 가능
- 카드 리스트(보유 카드) 화면 제공
- 카드 추가 버튼(+ UI)
- 등록된 카드 중 선택 후 결제 가능

### 3) 카드 정보 입력 요구사항
- 입력 항목:
  - 카드 번호
  - 만료일 (MM/YY)
  - 카드 소유자 이름
  - 보안코드(CVC/CVV)
  - 카드 비밀번호 일부 (UI 기준 2자리)

### 4) 민감 정보 보안 요구사항
- 카드번호 마스킹 정책
  - 카드번호 전체를 항상 노출하지 않음
  - 카드 리스트/결제 화면에서는:
```
1111 - 2222 - •••• - ••••
```

- 카드 뒷면 8자리
  - “카드 뒷면 8자리” = 카드번호 마지막 8자리
  - 별도의 입력값이 아니라
  - 마스킹 처리 대상 데이터
  - UI에서 직접 노출 금지

- 기타 보안
  - CVC/CVV 입력값 마스킹
  - 카드 비밀번호 입력값 마스킹

## 📋 기능 목록
### 1) 상품 목록 연동
```
<PayButton />
<ProductActions />
```
기능: 
  - 담기 / 결제 버튼 묶음

 
### 2) 카드 목록 화면
```
<CardListModal />
<CardItem />
<CardPreview />
<SelectCardToPayButton />
<AddCardEntryButton />
```
기능:
  - 보유 카드 리스트 표시
  - 카드번호 마스킹 표시
  - 카드 선택 후 결제

### 3) 카드 추가 입력
```
<AddCardModal />
<CardNumberInput />
<ExpiryInput />
<CardHolderInput />
<CvcInput />
<CardPasswordInput />
<CardLivePreview />
<SubmitCardButton />
```
기능:
  - 입력값 카드 미리보기 반영
  - 입력 완료 후 카드 저장

### 4) 데이터 처리 및 보안
```
- maskCardNumber()
- maskSensitiveDigits()
- validateCardFormat()
```
